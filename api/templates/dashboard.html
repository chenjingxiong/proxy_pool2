代理保鲜与选取策略

配置代理池的保鲜（重新验证）间隔和选取权重。权重越大，该因素在选取时的影响越强。
保鲜间隔修改后需重启 scheduler 容器生效；权重保存后立即生效。

调度器每 N 分钟重新验证池中所有代理
最近验证通过的代理更优先（指数衰减）
响应速度越快的代理更优先

选取策略说明

加权选取：不再随机选取代理，而是根据综合得分加权随机。

鲜活度：使用指数衰减计算，刚验证的代理得分为 1.0，1 分钟后降至约 0.37，5 分钟后降至约 0.01。确保优先选择新鲜验证通过的代理。

速度：代理响应时间（秒）的倒数，越快得分越高。归一化到 [0, 1]。

综合得分 = 鲜活度权重 × 鲜活度分 + 速度权重 × 速度分

过滤：仅从 last_status=True（最近验证通过）的代理中选取。
